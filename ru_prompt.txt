Ты — эксперт по машинному обучению и Python. Твоя задача — написать полноценное приложение для анализа датасета InSDN (SDN Intrusion Dataset) в формате CSV. Датасет содержит три файла: Normal_data.csv, OVS.csv, metasploitable-2.csv. Каждый файл имеет более 80 числовых признаков (полученных CICFlowMeter) и метку класса (type или label: норма или тип атаки). Требования:


1. Загрузка и объединение данных:
  - Загрузить все три CSV-файла из указанной папки.
  - Определить целевую переменную: для OVS.csv и metasploitable-2.csv колонка 'type' содержит категории атак (DoS, DDoS, Probe, brute-force-attack, Web_attack, Botnet, Exploitation). Для Normal_data.csv все записи относятся к нормальному трафику.
  - Сформировать единый DataFrame с колонками: все признаки (числовые) + метка 'label' (0 — норма, 1 — атака) и 'attack_type' (тип атаки, для нормальных — 'benign').
  - Выполнить предобработку: обработка пропусков (замена на 0 или медиана), удаление константных признаков, при необходимости — масштабирование (StandardScaler).


2. Разделение данных:
  - Стратифицированное разделение на train (60%), validation (20%), test (20%) с учётом метки 'attack_type' (чтобы все типы атак были в каждой выборке).


3. Обучение встроенных моделей (бинарная классификация норма/атака):
  - Обучить как минимум три модели: LogisticRegression, RandomForestClassifier, GradientBoostingClassifier (или XGBoost, если установлен).
  - Для каждой модели выполнить подбор гиперпараметров с помощью GridSearchCV (ограниченный поиск) на валидационной выборке.
  - Оценить на тестовой выборке: accuracy, precision, recall, f1-score, ROC-AUC, confusion matrix (в виде графика и текста).


4. Многоклассовая классификация (тип атаки):
  - Обучить RandomForestClassifier (или любую другую) для предсказания типа атаки (только на атакующих образцах, либо на всех, где атака).
  - Метрики: classification_report (precision, recall, f1 per class), матрица ошибок.


5. Сохранение результатов в удобочитаемом виде:
  - Сохранить предсказания всех моделей на тестовой выборке в CSV-файл: 'predictions.csv' с колонками — реальная метка (binary, multiclass), предсказания каждой модели.
  - Сохранить метрики (бинарные и многоклассовые) в JSON-файл 'metrics.json'.
  - Сохранить обученные модели (через joblib или pickle).


6. Структура приложения:
  - Оформить как скрипт с функциями: load_data(), preprocess(), train_binary_models(), train_multiclass_model(), evaluate(), save_results().
  - В конце файла __main__ должен запускать весь пайплайн, обрабатывая аргументы командной строки: --data_dir (путь к CSV).
  - Добавить подробные комментарии и docstrings.
  - Использовать библиотеки: pandas, numpy, scikit-learn, matplotlib, seaborn, joblib, json, argparse.


7. Воспроизводимость:
  - Зафиксировать random_state=42 везде, где есть случайность.


8. Дополнительные пожелания:
  - Вывести сводку по распределению классов.
  - Визуализировать confusion matrix для лучшей бинарной модели и для многоклассовой.
  - Если какой-то CSV отсутствует, выдать понятную ошибку (проверка наличия входных данных).
